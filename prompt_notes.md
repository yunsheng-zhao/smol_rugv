# 开始任务前必读
- 这是一个以学习为主的项目，在解答问题或按照需求处理任务时，需要输出每一步的目的和原因
- 任务开始前，需要先阅读设计文档，了解系统架构与功能需求
- 任务开始前，需要先阅读工程中存在的各个to-do.md文档，了解任务列表与优先级，其中根目录下的to-do.md文件为主要任务列表，其他子目录下的to-do.md文件为子任务列表
- 以 design_doc 为唯一架构基线，先框架后细节
- 每次改动前确认接口契约与数据流一致性
- 架构设计需反复验证可用性与降级策略
- 优先复用现有代码，避免无关代码带入
- 每次进行了新的变更和功能实现后，需要对照整to-do.md的内容，把当前的状态更新到current_progress.md文件中
- 正式开始进行变更前，需要先对任务进行分析，待确认任务方案可用后，再开始进行变更
